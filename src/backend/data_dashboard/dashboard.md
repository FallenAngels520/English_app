设计成一个运营 + 增长 + 会员转化 + 内容效果 + 系统成本都能覆盖的版本，并且把每个指标的口径（怎么算）和建议图表/筛选也写清楚

## 1) 看板整体结构（推荐 5 个页签）
1、总览（Executive）：核心 KPI 一屏看清
2、用户增长（Growth）：新增、活跃、留存、回流
3、会员与收入（Monetization）：会员数、转化、续费、收入（如有）
4、学习与内容（Learning/Content）：学习行为、词库/课程效果、完成率
5、系统与成本（Ops/Cost）：接口成功率、耗时、模型调用、媒体存储成本

## 2) 总览页（最重要，一屏 12 个卡片 + 4 张趋势图）
### 顶部 KPI 卡片（建议）
 - DAU（日活）：当天有任意有效行为的去重用户数
 - WAU（周活）：过去 7 天去重用户数
 - MAU（月活）：过去 30 天去重用户数
 - 新用户（New Users）：当天注册/首次使用人数
 - 会员总数（Active Members）：当前有效期内会员数
 - 新增会员（New Members）：当天新开通会员数
 - 会员转化率（CVR）：当天开通会员人数 / 当天活跃人数
 - 续费率（Renewal Rate）：到期当日/周期内续费人数 / 到期人数
 - ARPU / ARPPU（可选）
 - 人均学习时长（Avg Study Minutes）
 - 人均练习次数（Avg Practice Count）
 - 活跃用户次日留存（D1 Retention）（或新用户 D1）

### 中部趋势图（建议 4 张）
1、DAU/MAU 趋势（折线，支持对比上周/上月）
2、新增用户 vs 新增会员（双轴）
3、会员转化漏斗（漏斗图）：访问/活跃 → 触达付费点 → 点击订阅 → 支付成功
4、学习完成度趋势（折线/堆叠）：完成任务人数、完成率

### 底部榜单（运营最爱）
 - Top 词单/课程（按完成人数、完成率、收藏）
 - Top 单词（按练习次数、错误率）
 - Top 渠道（如果你有渠道字段）

## 3) 指标口径（关键：统一定义，后面才不会吵）
### 3.1 活跃类（DAU/WAU/MAU）
活跃用户建议以“发生任一核心事件”定义，例如：
 - 发起一次 /chat（学习/记忆法生成）
 - 完成一次练习（practice）
 - 播放一次音频/查看单词详情（可选）

口径建议：去重 user_id，按本地时区（你可用 Europe/Berlin）切天。

### 3.2 会员数（至少分 3 个）
 - 有效会员数 Active Members：membership.status=active 且 expire_at >= now()
 - 累计会员数 Total Members：历史曾经付费过的去重用户数
 - 新增会员 New Members：某天 start_at 落在当天的人数（或支付成功当天）
再补两个运营常用：
 - 到期会员数 Expiring：未来 7 天内到期的人数
 - 流失会员 Churned：到期后 N 天未续费的人数

### 3.3 留存（强烈建议做 Cohort）
 - D1/D7/D30 留存（新用户留存）
    - cohort = 注册日（或首次活跃日）
    - retention = cohort 用户在第 N 天是否活跃过
 - 回流率
    - 过去 30 天未活跃，但今天活跃的人 / 今天活跃的人

### 3.4 学习与内容效果
 - 学习完成率：完成任务人数 / 参与任务人数
 - 单词掌握率（如果你有评分）：连续 X 次正确或熟练度超过阈值
 - 错误率：错误次数 / 总练习次数
 - 内容质量指标（对你生成内容很关键）：
    - mnemonic 采纳率（用户是否收藏/使用）
    - 图片/音频生成成功率
    - 生成耗时 P50/P95

### 3.5 成本与稳定性（如果你用大模型会非常有用）
 - 调用量：/chat 次数、生成图片次数、TTS 次数
 - 成功率：2xx / 总请求
 - 延迟：P50/P95/P99
 - 估算成本：tokens * 单价（先粗算也行）
 - 媒体存储增长：新增文件数、总大小、OSS 请求量（如有）

## 4) 增长页（Growth）应该怎么摆
### 图表区
 - 新增用户（折线/柱状）
 - DAU/WAU/MAU（折线，可叠加）
 - 活跃构成（堆叠柱）：新用户活跃 / 老用户活跃 / 回流用户
 - 留存矩阵（Cohort Heatmap）：注册周 × 第 N 周留存

### 细分筛选（一定要有）
 - 时间范围：7/30/90 天，自定义
 - 渠道：organic / campaign / referral（如果有）
 - 地区/语言（如果有）
 - 客户端：iOS/Android/Web
 - 版本号：app_version（排查版本问题很关键）


## 5) 会员页（Monetization）设计重点
### 核心模块
1. 会员漏斗
 - 活跃用户 → 打开付费页 → 点击订阅 → 支付成功 → 首次学习完成

2. 订阅结构
 - 月/季/年占比

3. 续费与流失
 - 到期人数、续费人数、续费率（按到期 cohort）

4. LTV（可选）
 - 注册 cohort 的累计付费金额曲线

### 会员指标建议
 - 付费渗透率：有效会员 / MAU
 - 首购转化率：首购人数 / 新用户（或活跃用户）
 - 退款率（如果有）
 - 客单价（AOV）：订单金额 / 订单数

## 6) 学习与内容页（Learning/Content）更“丰富”的做法
### 学习行为漏斗（强烈建议）
 - 活跃用户
 - 学习用户（进入学习页/开始任务）
 - 完成学习（完成当天任务）
 - 连续学习（连续 3 天/7 天）

### 内容维度排行

 - 词单：开始人数、完成率、平均用时、次日回访率
 - 单词：练习次数、错误率、平均掌握天数

### 生成内容（你产品特色）
 - mnemonic/图片/音频的成功率、平均耗时
 - 用户对生成内容的互动：收藏、复制、分享、重生成率（重生成高可能意味着不满意）

## 7) 数据实现建议（否则 DAU/会员/学习会缺口径）
如果你现在还没有统一埋点/事件表，我建议你加一张通用事件表，后续所有指标都能靠它：

### events

 - id

 - user_id（匿名也行：device_id/anonymous_id）

 - event_name：app_open / chat / practice_done / pay_success / subscribe_view …

 - event_time

 - properties_json：渠道、版本、课程、word_id、耗时等

这样 DAU/MAU、漏斗、留存、内容排行都好做，也方便后期接 BI。

另外会员建议单独表：

### memberships

 - user_id

 - plan_type（monthly/yearly）

 - start_at

 - expire_at

 - status

 - order_id（可选）
